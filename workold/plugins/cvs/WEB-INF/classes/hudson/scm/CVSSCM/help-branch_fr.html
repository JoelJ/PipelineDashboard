Si cette option est positionnée, Jenkins lancera CVS avec -r pour construire une branche particulière. Si cette option est laissée vide, Jenkins construira la branche principale (trunk).

Normalement, Jenkins lance cvs check-out et update avec l'option -D date pour être sûr de récupérer une image cohérente du repository et assurer une meilleure reproductibilité. Cela peut causer des problèmes sur le nom de la branche spécifié ici est en réalité le nom d'un tag. Indiquer à Jenkins que ce nom est un tag corrigerait ce problème.
